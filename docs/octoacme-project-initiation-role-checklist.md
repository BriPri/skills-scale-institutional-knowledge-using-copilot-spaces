# OctoAcme Project Initiation Role Checklist

## Purpose
This checklist ensures all roles are properly engaged during project initiation. Use it to verify that each role has completed their initiation responsibilities and that handoffs are clear.

---

## Project Sponsor Checklist

- [ ] Reviewed and approved the project charter / one-pager
- [ ] Confirmed project aligns with organizational strategy
- [ ] Committed resources (budget, personnel)
- [ ] Identified key stakeholders requiring executive engagement
- [ ] Scheduled initial steering committee or sponsor briefing
- [ ] Communicated project sponsorship to the organization

**Handoff to**: Project Manager (project authorization and resources confirmed)

---

## Project Manager Checklist

- [ ] Created or reviewed the project one-pager
- [ ] Established the project repository or board structure
- [ ] Drafted the initial stakeholder list and communication plan
- [ ] Identified high-level timeline and key milestones
- [ ] Created initial risk list
- [ ] Scheduled kickoff meeting with delivery team
- [ ] Confirmed team roles and availability
- [ ] Set up regular status reporting cadence

**Handoff to**: Delivery team (kickoff meeting scheduled, artifacts in place)

---

## Product Manager Checklist

- [ ] Defined the problem statement and business case
- [ ] Established success metrics and acceptance criteria
- [ ] Prioritized initial backlog or feature list
- [ ] Identified user personas and stakeholder needs
- [ ] Aligned with Project Sponsor on business outcomes
- [ ] Reviewed one-pager with Business Analyst

**Handoff to**: Business Analyst (requirements gathering can begin)

---

## Business Analyst Checklist

- [ ] Reviewed the project one-pager and problem statement
- [ ] Identified key stakeholders for requirements gathering
- [ ] Scheduled requirements workshops or interviews
- [ ] Drafted initial user stories or requirements outline
- [ ] Confirmed requirements traceability approach
- [ ] Aligned with QA Lead on acceptance criteria format

**Handoff to**: Developers and QA Lead (specifications ready for review)

---

## QA Lead Checklist

- [ ] Reviewed the project scope and success metrics
- [ ] Identified initial testing approach and strategy
- [ ] Reviewed acceptance criteria format with Business Analyst
- [ ] Assessed testing resource needs
- [ ] Established defect tracking and reporting process
- [ ] Confirmed Definition of Done with Project Manager

**Handoff to**: Developers (testing expectations established)

---

## Change Manager Checklist

- [ ] Identified potential organizational impacts of the project
- [ ] Assessed stakeholder readiness for change
- [ ] Developed initial change management approach
- [ ] Aligned with Communications Coordinator on messaging
- [ ] Identified training or support needs
- [ ] Documented initial change risks

**Handoff to**: Communications Coordinator (change communication plan aligned)

---

## Communications Coordinator Checklist

- [ ] Reviewed the stakeholder list and communication needs
- [ ] Drafted the project communication plan
- [ ] Established communication channels (email lists, Slack channels, etc.)
- [ ] Created initial status report template
- [ ] Coordinated with Change Manager on change announcements
- [ ] Scheduled first stakeholder update or town hall

**Handoff to**: All stakeholders (communication channels established)

---

## Developers Checklist

- [ ] Reviewed the project one-pager and technical scope
- [ ] Confirmed development environment and tooling
- [ ] Identified initial technical risks or dependencies
- [ ] Provided input on initial estimates
- [ ] Reviewed coding standards and contribution guidelines
- [ ] Confirmed branching and CI/CD setup

**Handoff to**: Project Manager (technical readiness confirmed)

## Decision Gate

**Move to Planning when:**
- [ ] All role checklists are complete
- [ ] Sign-off obtained from all required roles
- [ ] Kickoff meeting scheduled
- [ ] Initial artifacts created and accessible
